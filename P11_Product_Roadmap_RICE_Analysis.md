## RICE Framework Methodology

**RICE Score = (Reach × Impact × Confidence) / Effort**

- **Reach**: Number of clients/users affected per quarter (0-1000+)
- **Impact**: Massive (3), High (2), Medium (1), Low (0.5)
- **Confidence**: Percentage certainty (0-100%)
- **Effort**: Person-months required

**Priority Tiers:**
- **Tier 0 (Critical)**: RICE > 50 - Must complete for viability
- **Tier 1 (High)**: RICE 25-50 - Core differentiators
- **Tier 2 (Medium)**: RICE 10-25 - Competitive features
- **Tier 3 (Low)**: RICE < 10 - Nice-to-have

## Risk Analysis

### Technical Risks

**Risk:** Data integration complexity
- **Mitigation:** Start with 2-3 platforms, expand gradually
- **Contingency:** Build adapters for non-standard APIs

**Risk:** AI model accuracy/reliability
- **Mitigation:** Human-in-the-loop for high-stakes decisions
- **Contingency:** Maintain manual backup processes Q1-Q2

**Risk:** Platform API changes/deprecations
- **Mitigation:** Abstraction layer, monitor platform updates
- **Contingency:** Budget for urgent refactoring

### Business Risks

**Risk:** Client resistance to AI automation
- **Mitigation:** Emphasize human oversight, frame as "augmentation"
- **Contingency:** Offer hybrid service tiers

**Risk:** Competitive pressure/copycats
- **Mitigation:** Move fast, build defensible data moats
- **Contingency:** Patent key innovations, emphasize execution quality

**Risk:** Regulatory changes (AI, Fair Housing)
- **Mitigation:** Build compliance monitoring into products
- **Contingency:** Legal review before launches

### Execution Risks

**Risk:** Roadmap too ambitious
- **Mitigation:** Focus on Tier 0 and Tier 1 only in 2026
- **Contingency:** Extend timeline, reduce scope

**Risk:** Team capacity constraints
- **Mitigation:** Offshore team, prioritization discipline
- **Contingency:** Pause new client acquisition to focus on product

**Risk:** Founder/key person dependency
- **Mitigation:** Documentation, knowledge sharing, redundancy
- **Contingency:** Cross-training, backup ownership
